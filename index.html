RECENT DECODE HISTORY
01

Text → Bits

Your text is UTF-8 encoded into a binary string. “Hi” → 0100100001101001. Any Unicode character works, including emoji.

bitstream.js
02

Chunking + Header

The bit stream is split into 480-bit chunks. Each chunk gets an 8-bit sync preamble, frame ID, total frame count, and checksum — 32 bits of header total.

encoder.js
03

Error Correction

Every 32-bit block is tripled. On decode, a majority vote across 3 copies corrects any single corrupted copy — tolerating noisy captures far better than raw data alone.

ecc.js
04

Interleaving

Bits are shuffled column-major before display. A localized scan error like glare or blur spreads across the payload instead of wiping out one whole region.

interleave.js
05

Glyph Rendering

Every 6 bits becomes one glyph: 2 bits for shape, 2 bits for rotation, and 2 bits for fill density. That creates 64 distinct symbol states per cell.

glyph.js
06

Finder Patterns

Three bold corner squares anchor the code. The scanner uses them to locate, orient, and warp the image into a square before reading each cell.

finder.js · warp.js
07

Scanning

The camera or uploaded image is grayscale + thresholded, blobs are labeled, finder patterns are detected, and each cell’s density is classified back into bits.

preprocess.js · blob.js · classify.js
08

Stabilisation + Reassembly

A frame is only accepted after 2 identical consecutive reads. Frames are collected by ID; when all arrive, payloads are concatenated and decoded back to text.

stabilize.js · scanner.js
