01

Text → Bits

Your text is UTF-8 encoded into a binary string. "Hi" → 0100100001101001. Any Unicode character works.

bitstream.js
02

Chunking + Header

The bit stream is split into 480-bit chunks. Each chunk gets an 8-bit sync preamble, frame ID, total frame count, and a checksum — 32 bits of header total.

encoder.js
03

Error Correction

Every 32-bit block is tripled. On decode, a majority vote across 3 copies corrects any single corrupted copy — tolerating ~33% cell error rate.

ecc.js
04

Interleaving

Bits are shuffled column-major before display. A localised scan error (smudge, glare) will corrupt spread-out bits, making ECC far more effective.

interleave.js
05

Glyph Rendering

Every 6 bits becomes one glyph: 2 bits for shape (circle/square/triangle/diamond), 2 bits for rotation, 2 bits for fill density. 64 distinct states per cell.

glyph.js
06

Finder Patterns

Three bold bordered squares sit in the corners. They are not data — the scanner uses them to locate and orient the code, then warps it into a square for reliable cell reading.

finder.js · warp.js
07

Scanning

The camera feed is grayscale + adaptively thresholded. Connected components are labelled, the 3 largest square blobs are treated as finders, and each data cell's pixel density is classified.

preprocess.js · blob.js · classify.js
08

Stabilisation + Reassembly

A frame is only accepted after 2 identical consecutive reads (eliminates noise). Frames are collected by ID; when all arrive, payloads are concatenated and decoded back to text.

stabilize.js · scanner.js
